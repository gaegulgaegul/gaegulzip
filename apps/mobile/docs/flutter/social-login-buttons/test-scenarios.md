# 소셜 로그인 버튼 컴포넌트 테스트 시나리오

> 생성일: 2026-01-18
> 대상 기능: 카카오, 네이버, 애플, 구글 소셜 로그인 버튼 컴포넌트
> 참조 문서: brief.md, design-spec.md, user-stories.md

## 개요

4개 소셜 로그인 플랫폼(카카오, 네이버, 애플, 구글)의 버튼 컴포넌트를 표준 Material 위젯으로 구현했습니다. 각 플랫폼의 공식 디자인 가이드라인을 100% 준수하며, GetX 상태 관리를 사용합니다.

## 테스트 환경

- **플랫폼**: iOS, Android
- **Flutter 버전**: 3.0 이상
- **Dart 버전**: 3.0 이상
- **테스트 도구**:
  - FlutterTestMcp (자동화): `npx -y flutter-test-mcp`
  - @mobilenext/mobile-mcp (UI 검증): `npx -y @mobilenext/mobile-mcp`

## 사전 조건

- [ ] 앱이 빌드되고 실행 가능한 상태 (`cd apps/wowa && flutter run`)
- [ ] iOS 시뮬레이터 또는 Android 에뮬레이터가 실행 중
- [ ] melos bootstrap이 정상적으로 완료됨
- [ ] flutter_svg 패키지가 설치됨
- [ ] 로고 SVG 파일이 assets에 배치됨

## 테스트 시나리오: 로그인 화면

### Scenario 1: 카카오 로그인 (Happy Path)

**Given**: 로그인 화면이 로드되어 4개의 소셜 로그인 버튼이 표시된 상태

**When**: 카카오 로그인 버튼을 탭한다

**Then**:
- 카카오 버튼에 로딩 인디케이터 (CircularProgressIndicator)가 표시된다
- 버튼이 비활성화되어 추가 클릭 불가능하다
- 다른 소셜 버튼(네이버, 애플, 구글)은 정상 상태 유지
- 2초 후 성공 스낵바 "카카오 로그인 성공" 표시
- 로딩 인디케이터가 사라지고 버튼이 다시 활성화된다

**수동 테스트 절차**:
1. 앱 실행: `cd apps/wowa && flutter run`
2. 로그인 화면 확인 (4개 버튼: 카카오, 네이버, 애플, 구글)
3. 카카오 버튼 탭
4. 버튼 중앙에 검은색 CircularProgressIndicator 확인
5. 2초 대기
6. 하단에 녹색 성공 스낵바 "카카오 로그인 성공" 표시 확인
7. 스크린샷 캡처: before_tap.png, loading.png, success.png

```bash

- "카카오 계정으로 로그인 버튼을 탭한다"
- "카카오 버튼에 로딩 인디케이터가 표시되는지 확인한다"
- "2초 동안 기다린다"
- "로딩 인디케이터가 사라지는지 확인한다"
- "카카오 로그인 성공 메시지가 표시되는지 확인한다"
```

**UI 검증 (@mobilenext/mobile-mcp)**:

## 테스트 시나리오: 독립적인 로딩 상태

### Scenario 5: 각 버튼 독립적으로 로딩 (Alternative Path)

**Given**: 로그인 화면이 로드된 상태

**When**: 카카오 버튼을 탭하고, 로딩 중에 네이버 버튼을 탭한다

**Then**:
- 카카오 버튼만 로딩 인디케이터 표시
- 네이버 버튼은 정상 상태 유지 (탭 가능)
- 카카오 로딩 완료 후 네이버 버튼 탭 시 네이버만 로딩 표시
- 두 버튼의 로딩 상태가 독립적으로 관리됨

**수동 테스트 절차**:
1. 로그인 화면에서 카카오 버튼 탭
2. 즉시 네이버 버튼 탭 (카카오 로딩 중)
3. 카카오 버튼만 로딩 상태인지 확인
4. 네이버 버튼도 로딩 상태로 전환되는지 확인
5. 각 버튼이 독립적으로 성공 스낵바를 표시하는지 확인

**자동화 테스트 (FlutterTestMcp)**:

- "카카오 버튼에 로딩 인디케이터가 표시되는지 확인한다"
- "네이버 계정으로 로그인 버튼이 여전히 활성화되어 있는지 확인한다"

- "카카오와 네이버 로그인 성공 메시지가 모두 표시되는지 확인한다"

**검증 체크리스트**:

## 테스트 시나리오: 에러 처리

### Scenario 6: 네트워크 에러 (Error Case)

**Given**: 로그인 화면이 로드되고, 네트워크가 끊긴 상태

**When**: 카카오 로그인 버튼을 탭한다

**Then**:
- 카카오 버튼이 로딩 상태로 전환
- 2초 후 로딩 해제
- 빨간색 에러 스낵바 표시: "로그인 실패 - 네트워크 연결을 확인해주세요"
- 버튼이 다시 활성화되어 재시도 가능

**수동 테스트 절차**:
1. 디바이스의 Wi-Fi 및 모바일 데이터 비활성화
2. 로그인 화면에서 카카오 버튼 탭
3. 로딩 인디케이터 표시 확인
4. 2초 대기
5. 빨간색 에러 스낵바 확인 (하단)
6. 버튼이 다시 정상 상태로 돌아가는지 확인
7. Wi-Fi 재활성화 후 버튼 재탭 → 성공 확인

**자동화 테스트 (FlutterTestMcp)**:

- "카카오 계정으로 로그인 버튼을 탭한다"
- "카카오 버튼에 로딩 인디케이터가 표시되는지 확인한다"
- "2초 동안 기다린다"
- "에러 메시지 '네트워크 연결을 확인해주세요'가 표시되는지 확인한다"

### Scenario 7: 사용자 취소 (Edge Case)

**Given**: 로그인 화면에서 카카오 버튼을 탭한 상태

**When**: OAuth 화면에서 사용자가 "취소" 버튼을 누른다

**Then**:
- AuthException with code 'user_cancelled' 발생
- 에러 스낵바 표시하지 않음 (정상 플로우)
- 로딩 상태 해제

2. OAuth 화면에서 "취소" 버튼 탭
3. 에러 메시지가 표시되지 않는지 확인
4. 로그인 화면으로 복귀 확인
5. 카카오 버튼이 정상 상태인지 확인

**검증 체크리스트**:

- [ ] 에러 스낵바 미표시
- [ ] isLoading = false (finally 블록)
- [ ] 사용자 경험 자연스러움

### Scenario 8: 버튼 중복 클릭 방지 (Edge Case)

**Given**: 로그인 화면이 로드된 상태

**When**: 카카오 버튼을 연속으로 빠르게 3번 탭한다

**Then**:
- 첫 번째 탭: 로딩 상태로 전환
- 두 번째, 세 번째 탭: 무시됨 (onPressed = null 상태)
- 로딩 완료 후 한 번만 성공 스낵바 표시
- 중복 API 호출 없음

**수동 테스트 절차**:
1. 로그인 화면에서 카카오 버튼 빠르게 3번 연속 탭
2. 로딩 인디케이터가 한 번만 표시되는지 확인
3. 성공 스낵바가 한 번만 표시되는지 확인
4. 콘솔 로그에서 API 호출이 1번만 발생했는지 확인

**검증 체크리스트**:
- [ ] isLoading = true 시 onPressed = null
- [ ] InkWell의 onTap: isLoading ? null : onPressed
- [ ] 중복 호출 방지 확인

### Scenario 15: Hot Reload 정상 동작

**Given**: 앱이 실행 중이고 로그인 화면이 표시된 상태

**Then**:

- 로딩 상태가 초기화됨

**수동 테스트 절차**:
1. `cd apps/wowa && flutter run`
2. 로그인 화면 확인
3. 코드 수정 (예: 버튼 텍스트 변경)
4. 터미널에서 `r` 키 입력 (Hot Reload)
5. 화면이 즉시 업데이트되는지 확인
6. 버튼 탭 테스트 → 정상 동작 확인

**검증 체크리스트**:
- [ ] Hot Reload 성공
- [ ] UI 즉시 업데이트
- [ ] GetX Controller 정상 작동
- [ ] 로딩 상태 초기화
