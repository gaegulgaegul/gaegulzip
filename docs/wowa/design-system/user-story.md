# 사용자 스토리: Design System v2 (Frame0 시각 언어 일치)

## 개요

현재 디자인 시스템은 "모노크롬 스타일 전환" 작업을 99% 완료했으나, Frame0 프로덕트의 실제 시각 언어와 비교한 결과 핵심적인 불일치가 발견되었습니다. 사용자가 Frame0의 손그림 스타일을 기대하고 앱을 사용할 때, 일관된 시각적 경험을 제공하여 프로토타입임을 직관적으로 인식하고 편안하게 피드백을 제공할 수 있도록 개선합니다.

**비즈니스 가치**:
- Frame0 브랜드 정체성과 100% 일치하여 사용자 신뢰도 향상
- "프로토타입임을 알리는 디자인"이라는 Frame0의 핵심 철학 구현
- 누락된 핵심 컴포넌트 제공으로 개발 생산성 향상
- 코드 품질 개선으로 유지보수성 및 안정성 확보

## 사용자 스토리

### P0 — 핵심 시각 언어 일치

#### US-1: 따뜻한 크림색 배경으로 편안한 프로토타입 분위기 조성
- **As a** 앱 사용자
- **I want to** 화면 배경이 순수한 흰색이 아닌 따뜻한 크림/오프화이트 톤으로 표시되고
- **So that** 손그림 스케치 느낌이 강화되어 프로토타입임을 자연스럽게 인식할 수 있다

#### US-2: X-cross 이미지 플레이스홀더로 Frame0 시그니처 요소 제공
- **As a** 앱 개발자
- **I want to** 이미지가 없는 영역에 대각선 X 패턴 플레이스홀더를 표시할 수 있고
- **So that** Frame0의 가장 대표적인 시각 요소를 활용하여 브랜드 일관성을 유지할 수 있다

#### US-3: 파란색 액센트로 링크와 선택 상태 명확히 표현
- **As a** 앱 사용자
- **I want to** 클릭 가능한 링크와 선택된 요소가 파란색으로 표시되고
- **So that** 상호작용 가능한 요소를 직관적으로 인식하고 현재 선택 상태를 명확히 파악할 수 있다

### P1 — 컴포넌트 완성도 및 스타일 정합성

#### US-4: 둥근 캡슐형 버튼으로 부드러운 인터랙션 제공
- **As a** 앱 사용자
- **I want to** 버튼이 둥글고 부드러운 캡슐 형태로 표시되고
- **So that** 시각적으로 친근하고 터치하기 편안한 느낌을 받을 수 있다

#### US-5: 손글씨체로 일관된 스케치 느낌 제공
- **As a** 앱 사용자
- **I want to** 모든 텍스트가 손글씨체로 표시되고
- **So that** 손그림 스케치 스타일의 일관된 경험을 받을 수 있다

#### US-6: 탭 바로 화면 간 빠른 전환
- **As a** 앱 사용자
- **I want to** 화면 하단의 탭 바를 통해 주요 화면 간 빠르게 이동할 수 있고
- **So that** 앱의 핵심 기능에 즉시 접근할 수 있다

#### US-7: 하단 네비게이션 바로 모바일 앱 표준 경험 제공
- **As a** 앱 사용자
- **I want to** 화면 하단의 네비게이션 바에서 현재 위치와 이동 가능한 메뉴를 확인할 수 있고
- **So that** 모바일 앱의 익숙한 탐색 패턴으로 편리하게 사용할 수 있다

#### US-8: 아바타로 사용자 및 프로필 정보 시각화
- **As a** 앱 사용자
- **I want to** 사용자 프로필이나 댓글 작성자를 아바타 이미지로 확인할 수 있고
- **So that** 누가 작성했는지 빠르게 인식할 수 있다

#### US-9: 라디오 버튼으로 단일 선택 옵션 제공
- **As a** 앱 사용자
- **I want to** 여러 옵션 중 하나만 선택할 수 있는 라디오 버튼을 사용할 수 있고
- **So that** 설정이나 선택지를 명확하게 구분할 수 있다

#### US-10: 검색 입력 필드로 빠른 정보 탐색
- **As a** 앱 사용자
- **I want to** 검색 아이콘이 포함된 입력 필드에 키워드를 입력하여 원하는 정보를 찾을 수 있고
- **So that** 많은 데이터 중에서 필요한 항목을 신속하게 찾을 수 있다

#### US-11: 여러 줄 텍스트 입력 필드로 긴 내용 작성
- **As a** 앱 사용자
- **I want to** 긴 텍스트를 입력할 수 있는 여러 줄 입력 필드를 사용할 수 있고
- **So that** 피드백이나 메모를 충분히 작성할 수 있다

#### US-12: 구분선으로 콘텐츠 영역 명확히 분리
- **As a** 앱 사용자
- **I want to** 화면의 서로 다른 콘텐츠 영역이 구분선으로 나뉘어 표시되고
- **So that** 정보를 구조적으로 이해하고 가독성을 높일 수 있다

#### US-13: 숫자 입력 필드로 정확한 수치 입력
- **As a** 앱 사용자
- **I want to** 숫자만 입력할 수 있는 전용 입력 필드를 사용할 수 있고
- **So that** 수량, 나이, 금액 등의 정보를 정확하게 입력할 수 있다

#### US-14: 링크로 외부 콘텐츠 접근
- **As a** 앱 사용자
- **I want to** 텍스트 링크를 탭하여 관련 페이지나 외부 콘텐츠로 이동할 수 있고
- **So that** 추가 정보를 쉽게 확인할 수 있다

#### US-15: 앱 바로 현재 위치와 주요 액션 확인
- **As a** 앱 사용자
- **I want to** 화면 상단의 앱 바에서 현재 화면 제목과 뒤로가기, 설정 등의 버튼을 확인할 수 있고
- **So that** 현재 위치를 파악하고 빠르게 액션을 실행할 수 있다

### P2 — 보조 기능 및 통일성

#### US-16: 소셜 로그인 버튼도 스케치 스타일로 통일
- **As a** 앱 사용자
- **I want to** 소셜 로그인 버튼이 공식 브랜드 가이드라인이 아닌 손그림 스케치 스타일로 표시되고
- **So that** 로그인 화면에서도 일관된 Frame0 디자인 경험을 받을 수 있다

#### US-17: 다크모드에서 더 어두운 배경 제공
- **As a** 앱 사용자
- **I want to** 다크모드 활성화 시 배경이 더 어두운 네이비/차콜 톤으로 표시되고
- **So that** 야간이나 저조도 환경에서 눈의 피로를 줄이고 몰입감을 높일 수 있다

## 사용자 시나리오

### 시나리오 1: 앱 최초 실행 시 따뜻한 크림색 배경 경험 (P0-1)

1. 사용자가 앱을 처음 실행한다
2. 화면 배경이 순수 흰색(#FFFFFF)이 아닌 따뜻한 크림색(#F5F0E8~#FAF8F5)로 표시된다
3. 사용자는 손그림 종이 위에 스케치된 듯한 자연스러운 느낌을 받는다
4. 결과: 프로토타입임을 자연스럽게 인식하고 편안하게 피드백을 제공할 준비가 된다

### 시나리오 2: 이미지 업로드 전 X-cross 플레이스홀더 확인 (P0-2)

1. 사용자가 프로필 이미지 업로드 화면을 연다
2. 이미지가 없는 영역에 대각선 X 패턴 플레이스홀더가 표시된다
3. 사용자는 Frame0의 시그니처 디자인 요소를 인식하고 이미지 업로드 영역임을 직관적으로 이해한다
4. 결과: 브랜드 일관성이 유지되고 사용자가 다음 액션을 명확히 파악한다

### 시나리오 3: 링크 클릭 시 파란색 액센트 확인 (P0-3)

1. 사용자가 공지사항 화면에서 "자세히 보기" 링크를 확인한다
2. 링크 텍스트가 파란색으로 표시되어 클릭 가능함을 시각적으로 알린다
3. 사용자가 링크를 탭한다
4. 링크가 선택 상태로 변경되며 여전히 파란색이 유지된다
5. 결과: 상호작용 가능한 요소를 직관적으로 인식하고 현재 상태를 명확히 파악한다

### 시나리오 4: 둥근 캡슐형 버튼 터치 (P1-4)

1. 사용자가 "질문 제출" 화면을 연다
2. 하단의 제출 버튼이 모서리가 둥글고 부드러운 캡슐 형태로 표시된다
3. 사용자는 시각적으로 친근하고 터치하기 쉬운 느낌을 받는다
4. 사용자가 버튼을 탭한다
5. 결과: 부드러운 인터랙션 경험을 받고 액션 완료에 대한 확신을 얻는다

### 시나리오 5: 손글씨체로 일관된 경험 (P1-5)

1. 사용자가 홈 화면, 설정 화면, 알림 화면을 차례로 이동한다
2. 모든 화면의 텍스트가 손글씨체(PatrickHand)로 표시된다
3. 사용자는 화면 전환에도 불구하고 일관된 손그림 스케치 느낌을 받는다
4. 결과: Frame0 디자인 시스템의 통일성을 경험하고 브랜드 신뢰도가 향상된다

### 시나리오 6: 하단 탭 바로 빠른 화면 전환 (P1-6)

1. 사용자가 홈 화면을 보고 있다
2. 화면 하단에 탭 바가 표시되어 있다 (홈, 알림, 설정 등)
3. 사용자가 알림 탭을 탭한다
4. 화면이 즉시 알림 목록 화면으로 전환된다
5. 현재 선택된 탭이 시각적으로 강조 표시된다
6. 결과: 앱의 핵심 기능에 빠르게 접근하고 현재 위치를 명확히 인식한다

### 시나리오 7: 여러 옵션 중 라디오 버튼으로 하나 선택 (P1-9)

1. 사용자가 설정 화면에서 "알림 빈도" 옵션을 확인한다
2. "즉시", "1시간마다", "하루 1번" 세 가지 라디오 버튼이 표시된다
3. 사용자가 "1시간마다" 라디오 버튼을 탭한다
4. 기존 선택이 해제되고 새로운 선택이 활성화된다
5. 결과: 단일 선택 옵션을 명확하게 설정하고 현재 설정을 시각적으로 확인한다

### 시나리오 8: 검색 입력 필드로 박스 검색 (P1-10)

1. 사용자가 박스 가입 화면을 연다
2. 화면 상단에 돋보기 아이콘이 포함된 검색 입력 필드가 표시된다
3. 사용자가 "크로스핏 강남"을 입력한다
4. 입력 중에도 검색 아이콘이 유지되어 기능을 명확히 표시한다
5. 결과: 빠르게 원하는 박스를 찾고 가입할 수 있다

### 시나리오 9: 긴 피드백을 텍스트 에어리어에 작성 (P1-11)

1. 사용자가 질문 제출 화면을 연다
2. 본문 입력 영역이 여러 줄 텍스트 에어리어로 표시된다
3. 사용자가 긴 피드백 내용을 입력한다
4. 입력 필드가 자동으로 확장되어 내용을 모두 확인할 수 있다
5. 결과: 충분한 피드백을 작성하고 제출할 수 있다

### 시나리오 10: 구분선으로 콘텐츠 영역 분리 (P1-12)

1. 사용자가 WOD 상세 화면을 연다
2. WOD 제목 영역과 운동 목록 영역 사이에 구분선이 표시된다
3. 운동 목록과 댓글 영역 사이에도 구분선이 표시된다
4. 사용자는 각 콘텐츠 영역을 명확히 구분한다
5. 결과: 정보를 구조적으로 이해하고 가독성이 향상된다

### 시나리오 11: 숫자 입력 필드로 무게 입력 (P1-13)

1. 사용자가 운동 기록 화면에서 무게를 입력한다
2. 숫자 입력 필드가 표시되고 키보드가 숫자 패드로 자동 전환된다
3. 사용자가 "75"를 입력한다
4. 문자 입력이 차단되어 숫자만 입력된다
5. 결과: 정확한 수치 정보를 빠르게 입력할 수 있다

### 시나리오 12: 소셜 로그인 버튼도 스케치 스타일 적용 (P2-16)

1. 사용자가 로그인 화면을 연다
2. 카카오, 네이버, 구글, 애플 로그인 버튼이 표시된다
3. 버튼이 공식 브랜드 가이드라인 스타일이 아닌 손그림 스케치 스타일로 표시된다
4. 사용자는 로그인 화면에서도 일관된 Frame0 디자인을 경험한다
5. 결과: 브랜드 통일성이 유지되고 프로토타입임을 지속적으로 인식한다

### 시나리오 13: 다크모드에서 더 어두운 배경 경험 (P2-17)

1. 사용자가 야간에 앱을 사용한다
2. 시스템 설정이 다크모드로 되어 있다
3. 앱 배경이 더 어두운 네이비/차콜 톤(#1A1D29)으로 표시된다
4. 텍스트와 요소가 충분한 대비를 유지한다
5. 결과: 눈의 피로가 줄어들고 야간 사용 경험이 개선된다

## 인수 조건 (Acceptance Criteria)

### P0 — 핵심 시각 언어

#### 크림색 배경 (US-1)
- [ ] 라이트모드의 기본 배경색이 순수 흰색(#FFFFFF)이 아닌 크림/오프화이트 계열(#F5F0E8~#FAF8F5)로 변경된다
- [ ] 모든 화면에 일관되게 적용된다
- [ ] 카드나 모달 등 표면(surface) 색상도 동일한 크림 톤 계열로 조화롭게 설정된다

#### X-cross 이미지 플레이스홀더 (US-2)
- [ ] 이미지가 없는 영역에 대각선 X 패턴을 렌더링하는 위젯을 사용할 수 있다
- [ ] X 패턴의 선 두께와 색상을 커스터마이징할 수 있다
- [ ] 정사각형 및 직사각형 비율에서 모두 자연스럽게 표시된다
- [ ] 프로필 이미지, 썸네일, 배너 등 다양한 크기에 적용 가능하다

#### 파란색 액센트 (US-3)
- [ ] 링크 텍스트의 기본 색상이 파란색(#2196F3 또는 Frame0의 파란색 계열)으로 설정된다

## 엣지 케이스

### 색상 및 시각 요소
- **크림색 배경과 텍스트 대비 부족**: 텍스트 색상을 충분히 어둡게 조정하여 WCAG 2.1 AA 기준 충족
- **X-cross 플레이스홀더가 매우 작은 영역에 렌더링**: 최소 크기(예: 24x24) 이하에서는 단일 X 또는 아이콘으로 단순화
- **파란색 액센트와 배경색 대비 부족**: 파란색 톤을 조정하여 충분한 대비 확보
- **다크모드에서 파란색이 너무 밝음**: 다크모드 전용 파란색 톤(더 어두운 블루) 제공

### 컴포넌트 동작
- **TabBar에 탭이 1개만 있을 때**: 탭 바를 숨기거나 비활성화 상태로 표시
- **BottomNavigationBar에 6개 이상 항목 추가 시도**: 최대 5개 제한, 초과 시 경고 또는 에러 발생
- **Avatar 이미지 로딩 실패**: 플레이스홀더(이니셜 또는 기본 아이콘) 자동 표시
- **Radio 그룹에서 초기 선택 없음**: 모든 라디오 버튼이 미선택 상태로 표시
- **SearchInput에 매우 긴 텍스트 입력**: 입력 필드가 오버플로우되지 않고 스크롤 가능
- **TextArea 최대 글자 수 초과**: 입력 차단 또는 경고 메시지 표시
- **NumberInput에 문자 입력 시도**: 키보드 제한으로 입력 차단
- **Link URL이 유효하지 않음**: 에러 처리 및 사용자에게 안내 메시지 표시
- **AppBar 제목이 매우 긴 경우**: 말줄임(...) 처리 또는 두 줄로 자동 줄바꿈

### 폰트 및 타이포그래피
- **손글씨체 미지원 플랫폼**: 폴백 폰트(Roboto 등)로 자동 대체
- **손글씨체에서 특수문자 렌더링 오류**: 폴백 폰트로 특정 문자만 대체
- **매우 큰 폰트 크기 설정 시 레이아웃 깨짐**: 최대 크기 제한 또는 반응형 레이아웃 적용

## 비즈니스 규칙

### 색상 및 스타일
- **크림색 배경은 라이트모드의 기본값**이며, 사용자가 순수 흰색을 원하면 옵션으로 제공 가능 (MVP에서는 크림색만 제공)
- **파란색 액센트는 링크와 선택 상태 전용**이며, CTA 버튼 등 주요 액션은 별도 색상(코랄/오렌지) 사용
- **X-cross 플레이스홀더는 Frame0 시그니처 요소**로, 다른 디자인 시스템에서는 사용 제한

### 컴포넌트 사용 원칙
- **TabBar는 2~5개 탭**으로 제한 (모바일 화면 최적화)
- **BottomNavigationBar는 3~5개 항목** 권장 (iOS HIG 및 Material Design 기준)
- **Radio 그룹은 단일 선택 강제**, 다중 선택이 필요하면 Checkbox 사용
- **손글씨체는 UI 텍스트 전용**, 본문 콘텐츠나 긴 글은 Sans 폰트 권장

### 접근성 및 호환성
- **모든 색상 조합은 WCAG 2.1 AA 기준**을 충족해야 함
- **손글씨체가 로드 실패 시 폴백 폰트**로 즉시 대체
- **터치 타겟 크기는 최소 44x44pt** 유지

## 비기능 요구사항

### 성능
- **위젯 렌더링 시간**: 모든 신규 위젯이 16ms(60fps) 이내에 렌더링되어야 한다
- **X-cross 플레이스홀더 렌더링**: CustomPaint 기반으로 최적화하여 프레임 드롭 방지
- **TabBar/BottomNavigationBar 전환**: 애니메이션이 부드럽게(60fps) 실행되어야 한다

### 접근성
- **색상 대비**: 모든 텍스트와 배경의 대비가 WCAG 2.1 AA 기준(최소 4.5:1) 충족
- **터치 타겟 크기**: 모든 인터랙티브 요소가 최소 44x44pt 크기 유지
- **스크린 리더**: 모든 위젯에 의미 있는 시맨틱 레이블 제공
- **폰트 크기 확대**: 시스템 폰트 크기 설정에 반응하여 레이아웃 유지

### 에러 처리
- **이미지 로딩 실패**: X-cross 플레이스홀더 또는 기본 아이콘 자동 표시
- **폰트 로딩 실패**: 폴백 폰트로 즉시 대체, 사용자에게 노출하지 않음
- **잘못된 파라미터**: 기본값으로 안전하게 폴백, 개발 모드에서는 경고 로그 출력
- **네트워크 이미지 타임아웃**: 로딩 인디케이터 → 에러 플레이스홀더 순차 표시

- **미사용 파라미터 제거**: 모든 위젯에서 사용되지 않는 roughness/seed/bowing/enableNoise 파라미터 삭제
- **Public API 정리**: 내부 클래스(ColorSpec 등)를 private으로 변경
- **Deprecated API 교체**: withOpacity() → withValues(alpha:) 수정
- **Dropdown 외부 탭 처리**: Barrier 추가로 외부 탭 시 자동 닫힘 구현
