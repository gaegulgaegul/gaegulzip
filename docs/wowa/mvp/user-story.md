# 사용자 스토리: WOWA MVP (크로스핏 WOD 알리미)

## 개요

WOWA는 크로스핏 박스에서 WOD(Workout of the Day) 정보가 카카오톡, 인스타그램, 네이버 카페 등에 파편화되어 찾기 어렵고, 코치 간 전달 오류나 화이트보드 오타로 인해 혼란이 발생하는 문제를 해결합니다.

**핵심 가치**: 역할(코치/회원) 구분 없이 누구나 WOD를 등록할 수 있으며, 먼저 등록된 WOD가 기준(Base)이 되고 커뮤니티 합의를 통해 정확한 WOD를 확정합니다. 시스템은 옳고 그름을 판단하지 않으며, 사용자가 직접 선택합니다.

---

## 사용자 스토리

### US-1: 박스 가입 및 관리

**As a** 크로스핏 박스 회원
**I want to** 내가 다니는 박스를 검색하고 가입하거나, 없으면 새로 만들 수 있다
**So that** WOWA에서 우리 박스의 WOD를 관리하고 공유할 수 있다

### US-2: WOD 등록 (텍스트 입력)

**As a** 박스 구성원 (코치/회원 구분 없음)
**I want to** 오늘의 WOD를 텍스트로 입력하여 등록할 수 있다
**So that** 박스 구성원들이 오늘 어떤 운동을 하는지 알 수 있다

### US-3: Base WOD 자동 지정

**As a** 먼저 WOD를 등록한 사용자
**I want to** 내가 등록한 WOD가 자동으로 오늘의 기준 WOD(Base)가 되길 원한다
**So that** 다른 구성원들이 기본적으로 이 WOD를 보게 된다

### US-4: 오늘의 WOD 조회

**As a** 박스 구성원
**I want to** 오늘 등록된 WOD를 날짜별로 확인할 수 있다
**So that** 박스에 가기 전에 오늘의 운동을 미리 파악하고 준비할 수 있다

### US-5: Personal WOD 등록 (Base와 다른 경우)

**As a** 박스 구성원
**I want to** Base WOD와 다른 내용의 WOD를 발견했을 때 별도로 등록할 수 있다
**So that** 내가 본 실제 WOD를 기록하고, 최초 등록자에게 차이를 알릴 수 있다

### US-6: WOD 차이 알림 수신

**As a** Base WOD를 등록한 사용자
**I want to** 다른 사용자가 등록한 WOD가 내 WOD와 다를 때 알림을 받는다
**So that** 내가 등록한 WOD에 오류가 있는지 확인하고 수정할 수 있다

### US-7: WOD 변경 제안 및 승인

**As a** Base WOD 등록자
**I want to** Personal WOD를 확인하고, 그것이 더 정확하다면 Base로 승인할 수 있다
**So that** 박스 구성원들이 올바른 WOD를 기준으로 운동할 수 있다

### US-8: Base WOD 변경 시 알림

**As a** Base WOD로 운동을 계획한 사용자
**I want to** Base WOD가 변경되었을 때 알림을 받는다
**So that** 변경된 내용을 확인하고 운동 계획을 조정할 수 있다

### US-9: WOD 선택 및 기록

**As a** 박스 구성원
**I want to** Base WOD 또는 Personal WOD 중 내가 실제로 수행한 것을 선택할 수 있다
**So that** 내 운동 기록이 정확하게 남고, 이후 Base가 변경되어도 내 선택은 유지된다

### US-10: 단일 박스 정책

**As a** 사용자
**I want to** 한 번에 하나의 박스에만 소속될 수 있다
**So that** WOD 조회 시 혼란 없이 내 박스의 정보만 보게 된다

### US-11: 박스 변경 (자동 탈퇴)

**As a** 여러 박스를 이용하는 사용자
**I want to** 새 박스에 가입하면 기존 박스에서 자동으로 탈퇴된다
**So that** 항상 현재 박스의 WOD만 표시되어 혼란이 없다

## 사용자 시나리오

### 시나리오 1: 코치가 아침에 WOD를 먼저 등록 (정상 플로우)

1. 박스 코치가 오전 6시에 WOWA 앱을 열고 오늘의 WOD를 텍스트로 입력한다
   - WOD 타입: AMRAP 15분
   - 운동: Pull-up 10회, Push-up 20회, Air Squat 30회
2. 시스템은 이 WOD를 자동으로 **Base WOD**로 지정한다
3. 박스의 다른 구성원들에게 "오늘 WOD가 등록되었습니다" 알림이 전송된다
4. 회원들이 앱을 열어 오늘의 WOD를 확인한다
5. 결과: 박스 구성원들이 동일한 WOD를 보고 준비한다

### 시나리오 2: 회원이 화이트보드 오류를 발견하고 수정 제안 (핵심 플로우)

1. 회원 A가 오전 10시 수업에 참여하여 화이트보드를 본다
2. 화이트보드에는 "Push-up 15회"로 적혀 있는데, Base WOD는 "20회"로 등록되어 있다
3. 회원 A가 WOWA 앱에서 "새 WOD 등록"을 선택하고 화이트보드의 내용을 입력한다
   - Push-up 15회 (20회에서 변경)
4. 시스템은 Base WOD와 구조적으로 비교하여 차이를 발견하고, 자동으로 **Personal WOD**로 분류한다
5. Base WOD 등록자(코치)에게 "오늘 WOD가 다르게 등록된 것 같습니다" 알림이 전송된다
6. 코치가 알림을 확인하고 두 WOD를 비교한다
7. 코치가 "승인" 버튼을 누른다
8. 회원 A의 WOD가 새로운 **Base WOD**로 승격된다
9. 기존 Base WOD를 선택한 사용자들에게 "Base WOD가 변경되었습니다" 알림이 전송된다
10. 결과: 정확한 WOD가 Base가 되어 모든 구성원이 올바른 정보를 공유한다

### 시나리오 3: 코치가 변경 제안을 승인하지 않음 (대안 플로우)

1. 시나리오 2의 1~5단계 동일
2. 코치가 알림을 확인하고, "거부" 버튼을 누르거나 아무 응답도 하지 않는다
3. 기존 Base WOD는 그대로 유지된다
4. 회원 A의 Personal WOD는 계속 존재하며, 회원 A는 자신의 Personal WOD를 선택하여 기록할 수 있다
5. 결과: Base는 변경되지 않지만, 회원 A는 자신이 본 WOD를 선택할 수 있다

### 시나리오 4: 사용자가 운동 기록을 위해 WOD 선택

1. 회원 B가 오후 6시 수업 후 운동 결과를 기록하기 위해 앱을 연다
2. "오늘의 WOD" 화면에 Base WOD가 표시되고, 하단에 "다른 버전 보기" 버튼이 있다
3. 회원 B가 "다른 버전 보기"를 누르면 Personal WOD 목록이 표시된다
4. 회원 B가 Base WOD를 선택하고 "이 WOD로 기록" 버튼을 누른다
5. 시스템은 선택 시점의 WOD를 스냅샷으로 저장한다
6. 이후 Base WOD가 변경되어도, 회원 B의 기록은 선택 시점의 WOD로 유지된다
7. 결과: 사용자는 실제로 수행한 WOD를 정확하게 기록할 수 있다

### 시나리오 5: 박스 가입 및 변경

1. 신규 사용자가 WOWA 앱에 처음 로그인한다
2. "박스 선택" 화면이 표시되고, 검색창에 "크로스핏 강남"을 입력한다
3. 검색 결과에 "크로스핏 강남 (서울 강남구)" 박스가 표시된다
4. 사용자가 "가입하기" 버튼을 누른다
5. 가입이 완료되고 홈 화면으로 이동하여 오늘의 WOD를 확인한다
6. 3개월 후, 사용자가 이사를 가서 다른 박스를 이용하기 시작한다
7. 설정 화면에서 "박스 변경"을 선택하고, "크로스핏 판교"를 검색하여 가입한다
8. 시스템은 자동으로 "크로스핏 강남"에서 탈퇴 처리한다
9. 홈 화면에는 이제 "크로스핏 판교"의 WOD만 표시된다
10. 결과: 사용자는 항상 한 박스의 WOD만 보게 되어 혼란이 없다

### 시나리오 6: 박스가 검색에 없어 새로 생성

1. 사용자가 "박스 선택" 화면에서 "크로스핏 신촌"을 검색한다
2. 검색 결과에 해당 박스가 없다
3. "박스가 없으신가요? 새로 만들기" 버튼이 표시된다
4. 사용자가 버튼을 클릭하고 박스 정보를 입력한다
   - 이름: 크로스핏 신촌
   - 지역: 서울 서대문구
5. 시스템은 박스를 생성하고, 사용자를 자동으로 가입시킨다
6. 결과: 사용자는 자신의 박스를 만들고 즉시 사용할 수 있다

## 인수 조건 (Acceptance Criteria)

### 박스 관리
- [ ] 사용자는 박스 이름과 지역으로 검색할 수 있다
- [ ] 사용자는 검색 결과에서 박스를 선택하여 가입할 수 있다
- [ ] 사용자는 새 박스를 생성할 수 있다 (이름, 지역 입력)
- [ ] 박스 생성 시 생성자는 자동으로 해당 박스에 가입된다
- [ ] 사용자는 한 번에 하나의 박스에만 소속될 수 있다
- [ ] 새 박스에 가입하면 기존 박스에서 자동으로 탈퇴된다
- [ ] 사용자는 현재 소속된 박스 정보를 조회할 수 있다
- [ ] 박스의 구성원 목록을 조회할 수 있다

### WOD 등록 및 조회
- [ ] 사용자는 텍스트로 WOD를 입력하여 등록할 수 있다 (WOD 타입, 시간, 운동 목록, 반복 횟수)
- [ ] 박스 + 날짜별로 첫 WOD는 자동으로 Base WOD가 된다
- [ ] Base WOD 등록 시 박스의 모든 구성원에게 알림이 전송된다
- [ ] 사용자는 특정 날짜의 Base WOD를 조회할 수 있다
- [ ] 사용자는 오늘의 WOD를 조회할 수 있다
- [ ] Base WOD가 없는 날에는 "아직 등록된 WOD가 없습니다" 메시지가 표시된다

### Base/Personal WOD 분리
- [ ] 시스템은 새로 등록된 WOD를 Base WOD와 구조적으로 비교한다
- [ ] WOD 타입, 시간, 운동명, 반복 횟수가 다르면 Personal WOD로 자동 분류된다
- [ ] Personal WOD 등록 시 Base WOD 등록자에게 "WOD가 다르게 등록된 것 같습니다" 알림이 전송된다
- [ ] 사용자는 특정 날짜의 Base WOD와 Personal WOD 목록을 조회할 수 있다
- [ ] Personal WOD는 등록한 사용자 정보를 포함한다

### 변경 제안 및 승인
- [ ] Base WOD 등록자는 Personal WOD 알림을 받고 두 WOD를 비교할 수 있다
- [ ] Base WOD 등록자는 Personal WOD를 승인하여 Base로 변경할 수 있다
- [ ] Base WOD가 변경되면 기존 Base를 조회한 사용자들에게 알림이 전송된다
- [ ] Base WOD 등록자는 Personal WOD를 거부하거나 무응답할 수 있다
- [ ] 거부 또는 무응답 시 Base WOD는 그대로 유지된다
- [ ] Personal WOD는 승인 여부와 관계없이 계속 존재한다

### WOD 선택 및 기록
- [ ] 사용자는 Base WOD 또는 Personal WOD 중 하나를 선택할 수 있다
- [ ] WOD 선택 시 해당 시점의 WOD 내용이 스냅샷으로 저장된다
- [ ] 선택 후 Base WOD가 변경되어도 기존 선택 기록은 변경되지 않는다
- [ ] 사용자는 자신이 선택한 WOD 목록을 날짜별로 조회할 수 있다
- [ ] 사용자는 한 날짜에 한 번만 WOD를 선택할 수 있다

### 알림 시스템
- [ ] 박스 구성원은 박스 단위 알림을 수신할 수 있다 (Base WOD 등록)
- [ ] 특정 사용자는 개인 알림을 수신할 수 있다 (Personal WOD 차이 알림, Base 변경 알림)
- [ ] 알림은 모바일 푸시로 전송된다
- [ ] 사용자는 알림을 클릭하여 해당 WOD 화면으로 이동할 수 있다

## 엣지 케이스

### 박스 관리
- **중복 박스 생성 시도**: 같은 이름과 지역의 박스가 이미 존재하면 생성 대신 가입 안내 메시지 표시
- **박스 미가입 상태**: 박스에 가입하지 않은 사용자는 WOD 조회/등록 불가, "박스를 먼저 선택하세요" 메시지 표시
- **박스의 마지막 구성원 탈퇴**: 박스 구성원이 0명이 되어도 박스 데이터는 유지 (추후 재가입 가능)

### WOD 등록
- **동시 WOD 등록**: 두 사용자가 동시에 첫 WOD를 등록하면 시간순(created_at)으로 먼저 등록된 것이 Base가 됨
- **빈 필드 입력**: WOD 타입, 시간, 운동 목록은 필수 입력, 누락 시 "필수 항목을 입력하세요" 에러 메시지 표시
- **잘못된 WOD 형식**: 운동 횟수가 음수이거나, 시간이 0 이하이면 "올바른 값을 입력하세요" 에러 메시지 표시

### Personal WOD 분리
- **동일한 WOD 재등록**: Base WOD와 완전히 동일한 내용을 등록하면 Personal WOD로 분류되지 않고, "이미 등록된 WOD와 동일합니다" 안내 메시지 표시
- **운동명 표기 차이**: "Pull-up"과 "pullup"은 동일한 운동으로 처리 (정규화)
- **미세한 차이**: 운동 순서만 다르거나 휴식 시간이 다른 경우도 Personal WOD로 분류

### 변경 승인
- **승인 전 Personal WOD 등록자가 탈퇴**: Personal WOD는 유지되며 승인 가능 (등록자 표시만 "탈퇴한 사용자"로 표시)
- **Base WOD 등록자가 탈퇴**: 변경 제안 알림을 받을 수 없으므로, 박스 내 다른 구성원에게 승인 권한 이관 안내 (MVP에서는 승인 불가)
- **여러 Personal WOD 존재**: Base 등록자는 여러 Personal WOD 중 하나를 선택하여 승인 가능

### WOD 선택
- **선택 후 해당 WOD 삭제**: 선택 기록에는 WOD 스냅샷이 저장되어 있어 조회 가능
- **선택 없이 다음 날 넘어감**: 선택하지 않은 날짜는 기록 없음으로 표시
- **과거 날짜 WOD 선택**: 과거 날짜의 WOD도 선택 가능 (소급 기록)

### 알림
- **알림 수신 실패**: 디바이스 토큰이 유효하지 않으면 발송 실패, 재로그인 시 토큰 갱신
- **알림 과다 발송**: 같은 날짜에 여러 Personal WOD가 등록되면 Base 등록자에게 알림이 여러 번 발송될 수 있음 (의도된 동작)

## 비즈니스 규칙

### 권한 및 역할
- 시스템은 코치/회원 역할을 구분하지 않는다
- 모든 박스 구성원은 WOD를 등록, 조회, 선택할 수 있다
- Base WOD 등록자만 변경 제안을 승인/거부할 수 있다
- 시스템은 WOD의 옳고 그름을 판단하지 않는다

### 데이터 불변성
- 한 번 선택된 WOD 기록은 이후 Base WOD가 변경되어도 변경되지 않는다
- 선택 시점의 WOD 내용이 스냅샷으로 저장된다
- Personal WOD는 승인/거부 여부와 관계없이 삭제되지 않는다

### 박스 정책
- WOD는 반드시 박스 + 날짜 단위로 관리된다
- 사용자는 한 번에 하나의 박스에만 소속될 수 있다
- 새 박스 가입 시 기존 박스에서 자동으로 탈퇴된다

### 합의 모델
- 먼저 등록된 WOD가 자동으로 Base가 된다
- Base는 합의(승인)를 통해서만 변경된다
- 자동 승인은 없다

### 시간 및 날짜
- WOD는 날짜 단위로 관리되며, 서버 시간 기준으로 처리된다
- 하루에 여러 Base WOD가 있을 수 없다 (첫 등록만 Base)
- 과거 날짜의 WOD도 등록 및 선택 가능하다

## 비기능 요구사항

### 성능
- WOD 조회 응답 시간: 1초 이내
- WOD 등록 처리 시간: 2초 이내
- 알림 발송 지연 시간: 5초 이내
- 박스 검색 응답 시간: 1초 이내

### 접근성
- 모든 화면은 모바일 화면(320px 이상)에서 정상 동작해야 한다
- 버튼과 입력 필드는 터치하기 쉬운 크기(최소 44x44pt)여야 한다
- 텍스트는 읽기 쉬운 크기(최소 14pt)여야 한다
- 색상 대비는 WCAG 2.1 AA 기준을 준수한다

### 에러 처리
- 네트워크 오류 시: "인터넷 연결을 확인해주세요" 스낵바 메시지 표시
- 서버 오류 시: "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요" 메시지 표시
- 입력 유효성 오류 시: 해당 필드 하단에 구체적인 에러 메시지 표시
- 인증 토큰 만료 시: 자동으로 토큰 갱신 시도, 실패 시 로그인 화면으로 이동

### 데이터 보안
- 사용자 인증 토큰은 암호화된 저장소에 보관한다
- 사용자 개인정보는 서버와 통신 시 HTTPS를 사용한다
- 알림에는 민감한 개인정보를 포함하지 않는다

### 사용성
- 사용자는 3단계 이내로 원하는 기능에 도달할 수 있어야 한다
- 로딩 중에는 명확한 로딩 인디케이터를 표시한다
- 중요한 동작(박스 탈퇴, WOD 삭제 등)은 확인 다이얼로그를 표시한다
- 성공/실패 피드백은 즉시 제공한다 (스낵바, 모달 등)

### 확장성
- 박스당 최대 500명의 구성원을 지원한다
- 하루에 박스당 최대 50개의 Personal WOD를 지원한다
- 사용자당 최대 1년치 WOD 선택 기록을 보관한다

## 향후 확장 방향 (Out of Scope)

다음 기능들은 MVP에 포함되지 않으며, 사용자 피드백을 기반으로 우선순위를 결정합니다.

### 화이트보드 이미지 인식
- 사용자가 화이트보드 사진을 찍으면 OCR + AI가 자동으로 WOD를 구조화하여 입력

### 신뢰도 시스템
- 사용자별 신뢰도 점수를 계산하여 Base WOD 승인 시 참고
- 코치 배지 (선택적)로 코치 권위를 자연스럽게 표현

### 변경 히스토리 시각화
- WOD 변경 과정을 시간순으로 보여주는 타임라인

### 난이도 계산
- WOD의 운동 종류, 반복 횟수, 시간을 기반으로 참고용 난이도 표시

### 커뮤니티 기능
- 운동 결과 공유 및 코멘트
- 박스 구성원 간 랭킹 및 리더보드

### 멀티 박스 전환
- 여러 박스에 동시 소속 가능, 로그아웃 없이 박스 간 전환

### 웨어러블 연동
- Apple Watch, Garmin 등 웨어러블 기기와 실시간 운동 추적

### 커스텀 벤치마크
- 사용자 정의 벤치마크 WOD 추적 및 비교

## 성공 지표

MVP 출시 후 다음 지표를 모니터링하여 제품의 성공 여부를 판단합니다.

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| WOD 등록 후 조회율 | 70% 이상 | 등록된 WOD 수 대비 조회한 사용자 수 |
| Personal WOD 발생 비율 | 10~30% | Base WOD 대비 Personal WOD 등록 비율 |
| Base 변경 승인율 | 50% 이상 | 변경 제안 수 대비 승인된 제안 수 |
| 사용자 선택 완료율 | 80% 이상 | WOD 조회한 사용자 중 선택 완료한 비율 |
| 일일 활성 사용자 (DAU) | 박스당 30% 이상 | 박스 구성원 중 하루에 1회 이상 앱을 사용한 사용자 비율 |
| 사용자 유지율 (4주 후) | 50% 이상 | 가입 후 4주 뒤에도 앱을 사용하는 사용자 비율 |

## MVP 출시 기준 (Definition of Done)

다음 모든 조건이 충족되면 MVP를 출시합니다.

- [ ] 역할 구분 없이 누구나 WOD를 등록할 수 있다
- [ ] 박스 + 날짜별로 첫 WOD가 자동으로 Base WOD가 된다
- [ ] Base WOD와 다른 WOD는 자동으로 Personal WOD로 분류된다
- [ ] Base WOD 등록자는 Personal WOD를 승인하여 Base로 변경할 수 있다
- [ ] 사용자는 Base WOD 또는 Personal WOD 중 하나를 선택하여 기록할 수 있다
- [ ] 선택된 WOD는 이후 Base가 변경되어도 불변이다
- [ ] 사용자는 한 번에 하나의 박스에만 소속될 수 있다
- [ ] 박스 구성원에게 WOD 관련 알림이 전송된다
- [ ] 모든 인수 조건이 검증되었다
- [ ] 주요 엣지 케이스에 대한 처리가 구현되었다
